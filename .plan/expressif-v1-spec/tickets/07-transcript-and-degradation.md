# Pin transcript representation and text degradation

## Question

Every widget needs a canonical text form. Pin what the *model* sees in later turns after a widget rendered and the user interacted; what survives context compaction; what the transcript file records; and how the same agent session degrades when no expressif surface is attached (plain terminal fallback — the skill/plugin must not break the CLI experience). Decide whether the text form is generated by the plugin, the core, or specified per primitive.
